Now lets look at xml

XML is a markup language. XML stands for eXtensible Markup Language. Markup langages cannot be considered as programming languages

A markup language is slightly different from a programming language A programming language like C sharp, C++, Java, Kotlin, Python etc:- will allow you to define behaviors, interactions, and conditions. A markup language is used more to describe data Programming languages create dynamic interactions, whereas markup languages generally handle static things like user interfaces. Now we shall look at java and kotlin the programming languages primarily used by android app developers worldwide

Java is a high-level programming language originally developed by Sun Microsystems and now owned by the famous BaaS company oracle. It was originally designed for developing programs for set-top boxes and handheld devices, but later became a popular choice for creating web and android apps. The Java syntax is similar to C++, but is strictly an object-oriented programming language. In java there are more classes than any other programming language.

Kotlin is a newcommer in the world of high level programming languages. Kotlin is a general purpose, free, open source, programming language initially designed for the Java Virtual Machine and Android. Kotlin is a great language that combines object-oriented and functional programming features. It is focused on safety, clarity, and tooling support. Thats why tech giants and FAANG use kotlin for their android apps. It's very likely that Kotlin will be the primary programming language for Android in the future. However, while pretty much everyone agrees that Kotlin is the better language, most people , even experienced kotlin developers recommend to still start with Java as a beginner. But most consider for beginners C is the ideal choice.

Now backend.

When we come to backend, we have to know that database is the most important concept of backends. Most android apps use NoSql databases.

NoSql means not only SQL .SQL is Structured Query Language.Sql databases are usually relational tables. NoSql databases are non tabular, and store data differently than relational tables.

NoSQL databases come in a variety of types based on their data model. The main types are document, key-value, wide-column, and graph. They provide flexible schemas and scale easily with large amounts of data and high user loads.

Commonly the most used Nosql db is firebase - a cloud based NoSql BaaS. BaaS means Backend - as - a - Service. Firebase is a BaaS offered by google for developers to store and sync data realtime Firebase stores data in the format of JSON, a tree like structure made of curly braces. JSON stands for Java Script Object Notation

Javacript is another programming language which is used to write cloud functions to firebase servers. Google Cloud Functions is a serverless execution environment for building and connecting cloud services like firebase. With Cloud Functions one can write simple, single-purpose functions that are attached to events emitted from the cloud infrastructure and services. The function is triggered when the related event being watched is fired.

As we use the android framework for android apps, we need to learn some other important concepts of android

Manifest file:- Every app project must have an AndroidManifest.xml file at the root of the project. The manifest file describes essential information about your app to the Android build tools, the Android operating system, and Google Play.

Android studio:- Android Studio is the official integrated development environment for Google's Android operating system, built on JetBrains' IntelliJ IDEA software and designed specifically for Android development. A surprising fact is that android studio itself was written on java, kotlin and the evergreen c++

Gradle: Gradle is a build automation tool for multi-language software development. It controls the development process in the tasks of compilation and packaging to testing, deployment, and publishing.

SDKs: SDK stands for software development kit or devkit for short. SDK tools will include a range of things, including libraries, documentation, code samples, processes, and guides that developers can use and integrate into their own apps. SDKs are designed to be used for specific platforms or programming languages.

Everything in the world isn't perfect. Android also has some cons lets look at them

There are a many android devices with different resolutions and screen sizes to take care of. This makes the app design and UI development much more challenging.
